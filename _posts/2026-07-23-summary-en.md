## [SkewAdam cuts MoE optimizer memory by 97%](https://www.reddit.com/r/MachineLearning/comments/1v38k1m/skewadam_a_tiered_optimizer_that_cuts_moe_state/) ⭐️ 9.0/10

SkewAdam introduces a tiered optimizer state allocation for Mixture-of-Experts (MoE) models, reducing optimizer state memory from 50.6 GB to 1.29 GB (a 97.4% reduction) and enabling a 6.78B MoE model to fit on a single 40GB GPU without convergence loss. This breakthrough dramatically lowers the hardware barrier for training large MoE models, allowing researchers with consumer GPUs to experiment with state-of-the-art architectures. It addresses a critical memory bottleneck that previously required multi-GPU setups or high-memory enterprise hardware. SkewAdam allocates precision based on parameter behavior: backbone (5% of params) gets float32 momentum and factored second moment, experts (95%) get only factored second moment, and router (<0.01%) gets exact second moment. Peak training memory drops from 81.4 GB to 31.3 GB, and the optimizer is implemented as a single-file, dependency-free torch.optim.Optimizer.

reddit · r/MachineLearning · /u/Kooky-Ad-4124 · Jul 22, 07:04

**Background**: Mixture-of-Experts (MoE) models scale model capacity without proportionally increasing compute by using multiple specialized sub-networks (experts) activated per input. However, training MoEs is memory-intensive because standard optimizers like AdamW store per-parameter momentum and variance states, which can dominate the memory budget. SkewAdam's tiered approach exploits the observation that different parameter groups (backbone, experts, router) have different optimization needs, allowing aggressive memory savings without harming convergence.

## [GigaToken: ~1000x Faster Language Model Tokenization](https://github.com/marcelroed/gigatoken/) ⭐️ 8.0/10

GigaToken is a new open-source tokenizer that achieves approximately 1000x speedup over standard implementations by using SIMD optimization and caching techniques. While tokenization is a small fraction of inference time, this speedup is highly valuable for offline pre-training data preparation, where tokenizing terabytes of text can save significant time and cost. The performance gains come from heavily optimizing pretokenization (usually handled by a regex engine) using SIMD, minimizing branching, and caching pretoken mappings. Results are consistent across modern x86 and ARM CPUs and various tokenizers.

hackernews · syrusakbary · Jul 22, 17:20 · [Discussion](https://news.ycombinator.com/item?id=49010167)

**Background**: Tokenization is the process of converting text into tokens (subwords or characters) that language models can process. It is a fundamental step in NLP pipelines, often implemented using regex-based pretokenization followed by a lookup table. SIMD (Single Instruction, Multiple Data) allows CPUs to process multiple data elements in parallel, significantly accelerating pattern-matching tasks.
