Anthropic 发布了 Claude Sonnet 5，这是一个更快、更具代理能力的模型，能够规划、使用工具并自主运行。与更大的 Opus 模型相比，它提供了有竞争力的性能，但在成本效益方面存在微妙的权衡。 此次发布以更小、更快的模型推进了代理式 AI 能力，可能使自主 AI 助手更易获取。然而，成本与性能的权衡意味着用户必须根据具体用例在 Sonnet 5 和 Opus 之间谨慎选择。 在基准测试中，Sonnet 5 在 Terminal-Bench 上获胜，并在知识工作上与 Opus 持平，成本降低 40-60%，但 Opus 在编码、终端使用、计算机使用和推理方面小幅领先。社区基准测试显示 Sonnet 5 在常识问答、组合工具调用任务和谜题解决方面存在弱点。

一位开发者发现，Anthropic 的 Claude Code 工具在其 API 请求中嵌入了隐写标记，用于识别使用情况，很可能是为了检测中国公司的模型蒸馏行为，且未向用户透明披露。 这引发了对广泛使用的 AI 编码工具的重大透明度和信任问题，因为用户可能在不知情的情况下被以隐藏方式追踪请求。同时，这也凸显了保护专有模型与尊重用户隐私之间日益加剧的紧张关系。 这些隐写标记被嵌入发送到 Anthropic API 的请求文本中，不进行逆向工程很难检测。该技术让人联想到“隐蔽代码”，似乎旨在绕过可能剥离显式遥测数据的自定义 API 网关。

## [Ornith-1.0：开源 LLM 在智能体编程中达到 SOTA](https://simonwillison.net/2026/Jun/29/ornith/#atom-everything) ⭐️ 8.0/10

DeepReinforce 发布了 Ornith-1.0，一个用于智能体编程的开源权重 LLM 家族（MIT 许可证），参数规模从 9B 到 397B，基于 Gemma 4 和 Qwen 3.5（均为 Apache 2.0）。在编程基准测试中，它在同等规模的开源模型中达到了最先进的性能。 此次发布为自主编程代理提供了一个强大且许可宽松的替代方案，可能加速 AI 辅助软件开发。其自我脚手架训练方法——模型学习生成特定任务的脚手架——代表了一种新颖的范式，可能提升整个领域的智能体能力。 模型系列包括 9B 密集、31B 密集、35B MoE 和 397B MoE 变体，其中 35B GGUF（20GB）已可通过 LM Studio 使用。早期用户测试显示其能熟练进行多步工具调用，例如在代码库中查找特定函数。

rss · Simon Willison · 6月29日 16:17

**背景**: 智能体编程指的是 AI 代理自主规划、编写、测试和修改代码，只需极少的人工干预。传统的 LLM 编程助手依赖人类设计的脚手架来组织任务；Ornith-1.0 的自我脚手架创新允许模型在强化学习过程中生成自己的任务特定脚手架，从而提高解决方案质量。
